# Mari-Website online stellen (nur ansehen, ohne Zahlung)

Dieser Ordner enthält die fertige Landingpage als reine HTML-Dateien.
Startseite ist `index.html`. Du brauchst keinen Server – einfach hochladen.

> Hinweis: Der Button „Gratis testen" führt (noch) nicht zur echten Zahlung,
> weil dafür der Server + Stripe nötig wären. Alles andere ist voll sichtbar.
> Wenn du später wirklich Geld einnehmen willst, sag Bescheid – dann schalten
> wir den Server-Teil scharf.

---

## Weg A – GitHub Pages (empfohlen, du hast ja GitHub)

1. Auf https://github.com einloggen → oben rechts **+** → **New repository**.
2. Name z. B. `mari-website`, auf **Public** lassen, **Create repository**.
3. Auf der neuen Seite: **uploading an existing file** anklicken.
4. **Alle** Dateien aus diesem Ordner (inkl. der Datei `.nojekyll`) ins Feld ziehen
   → unten **Commit changes**.
5. Oben im Repo auf **Settings** → links **Pages**.
6. Bei „Source" **Deploy from a branch** wählen, Branch **main** / Ordner **/(root)**
   → **Save**.
7. Nach 1–2 Minuten erscheint oben deine Adresse, z. B.
   `https://DEIN-NAME.github.io/mari-website/` – das ist dein Live-Link.

## Weg B – Noch schneller ohne GitHub: Netlify Drop

1. https://app.netlify.com/drop öffnen.
2. Diesen kompletten Ordner per Drag & Drop ins Fenster ziehen.
3. Sofort bekommst du einen Live-Link (z. B. `https://zufallsname.netlify.app`).
   (Zum dauerhaften Behalten kurz kostenlos anmelden.)

---

## Später: eigene Domain

Beide Wege erlauben es, eine eigene Domain (z. B. `telequest.at`) zu hinterlegen –
bei GitHub Pages unter Settings → Pages → „Custom domain",
bei Netlify unter „Domain settings". Sag einfach Bescheid, wenn es soweit ist.
